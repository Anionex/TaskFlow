# 介绍
这是用于 https://github.com/Anionex/ 的项目的前端设计约束，目标是避开千篇一律的前端风格、完成富有空气感和现代saas风格的前端设计。

## 前端设计原则

1. 去除图标的滥用，尽量改成无图标的形式，或者是改成文字式的图标:
- 全局禁止使用 emoji 图标
- 图标是"文字的一部分"，和标题同级渲染，不用色块、圆角容器去单独框出来
- 避免任何多余的视觉容器——如果一个元素不需要边界来表达含义，就不给它边界
- 整体是一个统一的排版系统，而不是"内容 + 装饰"的拼凑

2. 去除色块滥用
- 对于前端区域上的视觉划分，不要用色块区分的形式，而是用轻边框的形式，避免视觉上的笨重感

3. 避免过度分层
- 只做必要的区域、按钮划分，如果不是必须，去除边框、边线，不要让前端变成格子间。

4.字体
- Google Fonts作为字体库。前期测试可直接通过CDN先引入。

5. 动画
- 使用GSAP作为动画库. 禁止炫技动画，保留最简单的交互反馈。前期测试可直接通过CDN先引入，当进入构建发布流程后去除。
- 按钮悬浮时，默认不使用任何浮动效果动画。
