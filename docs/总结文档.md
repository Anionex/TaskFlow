## 1. 需求文档

### 1.1 项目背景

在日常学习和工作中，人们需要管理大量的任务和待办事项。传统的纸质记录方式效率低下，而现有的任务管理工具往往过于复杂或需要付费。因此，我们决定开发一款轻量级、易用且免费的任务管理系统。

### 1.2 目标用户

- 学生：管理课程作业、考试复习、社团活动等

- 上班族：管理日常工作任务、项目进度等

- 个人用户：管理生活琐事、个人目标等

### 1.3 功能需求

#### 1.3.1 用户管理

- 用户注册：支持手机号注册，验证手机号格式（11位数字）

- 用户登录：支持手机号+密码登录，提供详细的错误提示

- 会话管理：基于Session的用户认证机制

- 修改密码：支持修改登录密码，需验证原密码

#### 1.3.2 任务管理

- 创建任务：支持设置任务标题、描述、分类、星级、截止日期

- 编辑任务：支持修改任务的各项属性

- 删除任务：支持删除任务并移至回收站

- 完成任务：支持标记任务为已完成/未完成状态

- 任务分类：支持按分类筛选任务（工作、学习、生活、其他）

- 任务排序：支持按创建时间、截止日期、星级排序

- 任务搜索：支持按关键词搜索任务

#### 1.3.3 批量操作

- 批量选择：支持选择多个任务进行批量操作

- 批量删除：支持一次性删除多个选中的任务

- 清空功能：支持一键清空待办、已完成、已过期列表

#### 1.3.4 任务统计

- 任务概览：显示总任务数、待办数、已完成数

- 饼图统计：显示已完成、待做、已过期任务的比例

- 柱状图统计：显示最近12个月的任务完成量

#### 1.3.5 回收站

- 任务恢复：支持从回收站恢复误删的任务

- 永久删除：支持永久删除回收站中的任务

- 清空回收站：支持一键清空回收站

#### 1.3.6 自动生成任务

- 模板管理：支持创建、编辑、删除任务模板

- 自动生成：支持按日、周、月自动生成任务

- 灵活配置：支持设置生成时间、截止时间等

#### 1.3.7 每日打卡

- 签到功能：支持每日签到打卡

- 连续统计：显示当前连续签到天数和最长连续天数

- 打卡激励：签到后显示励志名言

#### 1.3.8 数据管理

- 数据导出：支持将任务数据导出为ZIP文件

- 数据导入：支持从ZIP文件导入任务数据

#### 1.3.9 主题切换

- 普通模式：默认的浅色主题

- 护眼模式：低蓝光的护眼主题

- 夜间模式：深色背景的夜间主题

### 1.4 非功能需求

#### 1.4.1 性能需求

- 页面加载时间不超过3秒

- API响应时间不超过500毫秒

- 支持同时处理多个用户请求

#### 1.4.2 安全需求

- 密码使用SHA-256加密存储

- 使用Session机制进行用户认证

- 防止SQL注入和XSS攻击

#### 1.4.3 可用性需求

- 界面简洁美观，操作直观

- 支持响应式设计，适配移动端

- 提供友好的错误提示信息

#### 1.4.4 可维护性需求

- 代码结构清晰，模块化设计

- 添加必要的注释和文档

- 使用版本控制工具管理代码

#### 2.2.1 后端技术

- **UUID生成：** Uuid

- **HTML5：** 页面结构

- **CSS3：** 样式设计，支持主题切换

- **存储方式：** JSON文件存储

#### 2.3.2 任务模型 (Task)

struct Task {
    id: String,              // 任务ID (UUID)
    title: String,           // 任务标题
    description: String,     // 任务描述
    completed: bool,         // 是否完成
    category: String,        // 分类 (工作/学习/生活/其他)
    star_rating: u8,         // 星级 (0-5)
    start_date: String,      // 开始日期
    deadline: String,        // 截止日期
    created_at: String,      // 创建时间
    completed_at: String,    // 完成时间
}

#### 2.3.3 任务模板模型 (TaskTemplate)

struct TaskTemplate {
    id: String,              // 模板ID
    title: String,           // 任务标题
    description: String,     // 任务描述
    category: String,        // 分类
    star_rating: u8,         // 星级
    frequency: String,       // 频率 (daily/weekly/monthly)
    generate_day: u8,        // 生成日
    generate_time: String,   // 生成时间
    deadline_day: u8,        // 截止日
    deadline_time: String,   // 截止时间
    created_at: String,      // 创建时间
    last_generated: Option<String>, // 上次生成日期
}

#### 2.3.4 回收站模型 (RecycleBinItem)

struct RecycleBinItem {
    task: Task,              // 任务对象

struct CheckinData {

## 3. API文档

### 3.1 基础信息

- **Base URL：** `http://localhost:8080/api`

- **请求格式：** JSON

- **响应格式：** JSON

- **认证方式：** Session ID (通过 `X-Session-Id` 请求头传递)

### 3.2 通用响应格式

{
    "success": true/false,
    "message": "提示信息",
    "data": {} // 可选，返回数据
}

### 3.3 用户认证 API

#### 3.3.1 用户注册

- **URL：** `POST /register`

- **请求体：**

{
    "phone": "13800138000",
    "password": "123456"
}

- **响应：**

- 成功：`{"success": true, "message": "注册成功"}`

- 失败：`{"success": false, "message": "错误信息"}`

#### 3.3.2 用户登录

- **URL：** `POST /login`

- **请求体：**

{
    "phone": "13800138000",
    "password": "123456"
}

- **响应：**

- 成功：`{"success": true, "message": "登录成功", "data": "session_id"}`

- 失败：

- 手机号格式错误：`{"success": false, "message": "手机号格式错误，应为11位数字"}`

- 手机号未注册：`{"success": false, "message": "该手机号未注册"}`

- 密码错误：`{"success": false, "message": "密码错误"}`

#### 3.3.3 用户登出

- **URL：** `POST /logout`

- **请求头：** `X-Session-Id: {session_id}`

- **响应：** `{"success": true, "message": "已退出登录"}`

#### 3.3.4 检查会话

- **URL：** `GET /session`

- **请求头：** `X-Session-Id: {session_id}`

- **响应：**

- 有效：`{"success": true, "message": "已登录", "data": "手机号"}`

- 无效：`{"success": false, "message": "未登录"}`

### 3.4 任务管理 API

#### 3.4.1 获取任务列表

- **URL：** `GET /tasks`

- **请求头：** `X-Session-Id: {session_id}`

- **查询参数：**

- `sort_by`：排序方式 (created/deadline/star)

- `category`：分类筛选

- `search`：搜索关键词

- `page`：页码

- `per_page`：每页数量

- **响应：**

{
    "success": true,
    "message": "获取成功",
    "data": {
        "items": [...],
        "total": 100,
        "page": 1,
        "per_page": 20,
        "total_pages": 5
    }
}

#### 3.4.2 创建任务

- **URL：** `POST /tasks`

- **请求头：** `X-Session-Id: {session_id}`

- **请求体：**

{
    "title": "完成作业",
    "description": "完成Rust大作业",
    "category": "学习",
    "star_rating": 3,
    "start_date": "2025-06-20",
    "deadline": "2025-06-25T18:00"
}

- **响应：**

{
    "success": true,
    "message": "任务创建成功",
    "data": {
        "id": "uuid",
        "title": "完成作业",
        ...
    }
}

#### 3.4.3 更新任务

- **URL：** `PUT /tasks/{id}`

- **请求头：** `X-Session-Id: {session_id}`

- **请求体：** (只传需要更新的字段)

{
    "title": "新标题",
    "star_rating": 5
}

#### 3.4.4 删除任务

- **URL：** `DELETE /tasks/{id}`

- **请求头：** `X-Session-Id: {session_id}`

- **响应：** `{"success": true, "message": "任务已移到回收站"}`

#### 3.4.5 切换任务状态

- **URL：** `POST /tasks/{id}/toggle`

- **请求头：** `X-Session-Id: {session_id}`

- **响应：** `{"success": true, "message": "任务已完成"}`

#### 3.4.6 批量删除任务

- **URL：** `POST /tasks/batch-delete`

- **请求头：** `X-Session-Id: {session_id}`

- **请求体：**

{
    "task_ids": ["id1", "id2", "id3"]
}

- **响应：** `{"success": true, "message": "已删除3个任务"}`

#### 3.4.7 清空任务

- **URL：** `DELETE /tasks/clear?status={status}`

- **请求头：** `X-Session-Id: {session_id}`

- **查询参数：** `status` (pending/completed/expired)

- **响应：** `{"success": true, "message": "已清空5个任务"}`

#### 示例3：使用Claude生成Chart.js图表代码

提示词：

请帮我使用Chart.js生成一个饼图，显示任务的完成状态分布。
需要显示：已完成、待做、已过期三个状态的数量和百分比。

AI生成的代码：

function renderPieChart(stats) {
    const ctx = document.getElementById("pie-chart").getContext("2d");
    const labels = ["已完成", "待做中", "已过期"];
    const values = [stats.completed, stats.pending, stats.expired];
    const colors = ["#27ae60", "#667eea", "#e74c3c"];
    
    pieChart = new Chart(ctx, {
        type: "pie",
        data: {
            labels: labels,
            datasets: [{
                data: values,
                backgroundColor: colors,
                borderWidth: 2,
                borderColor: "#ffffff"
            }]
        },
        options: {
            responsive: true,
            plugins: {
                tooltip: {
                    callbacks: {
                        label: function(context) {
                            const total = context.dataset.data.reduce((a, b) => a + b, 0);
                            const percentage = ((context.parsed / total) * 100).toFixed(1);
                            return `${context.label}: ${context.parsed} (${percentage}%)`;
                        }
                    }
                }
            }
        }
    });
}
